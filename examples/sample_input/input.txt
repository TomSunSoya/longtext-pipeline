Priya Patel [02:00 PM]: Build a real-time notification system. We have 2.5 million 
active users, peak concurrency around 400,000. Walk me through your approach.

Michael Torres [02:02 PM]: Let me clarify requirements:
1. Expected notification volume?
2. Strict ordering or eventual consistency?
3. What channels - in-app, email, SMS, push?
4. Latency requirements?

Priya Patel [02:03 PM]: Average 20 notifications/user/day with spikes. Order matters 
within session. All channels. In-app within 2 seconds, email/SMS within 5 minutes.

Michael Torres [02:05 PM]: I'd propose event-driven architecture with pub-sub:

**Core Architecture:**
- Multiple event sources and consumers
- Message broker: Kafka or AWS SNS/SQS

**Storage:**
- PostgreSQL for user preferences and metadata
- Redis for caching
- Possibly DynamoDB for history

Priya Patel [02:11 PM]: How do you handle in-app real-time vs batched email/SMS?

Michael Torres [02:12 PM]: Separate pipelines:

**In-App (Real-time):**
- WebSocket connections for active users
- Notification fanout service
- Server-Sent Events as fallback
- Store notifications for offline users

**Email/SMS (Batched):**
- Accumulate in queue
- Scheduled job every 5 minutes
- Group into digest emails
- Exponential backoff for failures

Priya Patel [02:16 PM]: What if a post is deleted before notification is delivered?

Michael Torres [02:17 PM]: Three safeguards:
1. **Event Validation**: Check content exists before delivery
2. **Event Sourcing**: Keep immutable log for audits
3. **Cancellation Mechanism**: Remove pending notifications when content deleted

## 2.2 Coding Exercise

Priya Patel [02:22 PM]: Implement rate limiter. User can trigger at most N 
notifications per minute. Excess should be queued.

Michael Torres [02:24 PM]: I'll use sliding window counter approach:

```python
import time
from collections import defaultdict

class RateLimiter:
    def __init__(self, max_requests: int, window_seconds: int):
        self.max_requests = max_requests
        self.window_seconds = window_seconds
        self.requests = defaultdict(list)
    
    def is_allowed(self, user_id: str):
        now = time.time()
        window_start = now - self.window_seconds
        
        self.requests[user_id] = [
            t for t in self.requests[user_id] if t > window_start
        ]
        
        if len(self.requests[user_id]) < self.max_requests:
            self.requests[user_id].append(now)
            return True
        return False
```

Michael Torres [02:28 PM]: For production, I'd use Redis with Lua scripts for 
atomicity across multiple servers. Trade-offs:
- **Advantages**: Consistent, durable, atomic
- **Disadvantages**: Added latency, single point of failure, cost
