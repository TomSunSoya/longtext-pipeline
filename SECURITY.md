# Security Policy

## Supported versions

This project is pre-1.0. Security fixes are applied to the latest `master` branch and the most recent tagged release, if one exists.

| Version | Supported |
|---------|-----------|
| `master` | Yes |
| Latest release | Yes |
| Older unreleased snapshots | No |

## Reporting a vulnerability

Please do not open a public GitHub issue for sensitive vulnerabilities.

Use one of these private channels instead:

1. GitHub private vulnerability reporting, if it is enabled for the repository.
2. A private message to the maintainer via GitHub: <https://github.com/TomSunSoya>

When reporting, include:

- The affected version or commit SHA
- A minimal reproduction or proof of concept
- Impact assessment
- Any suggested mitigation, if you have one

You can still use public issues for non-sensitive hardening ideas such as dependency bumps, safer defaults, or documentation improvements.

## What to expect

- Initial acknowledgement target: within 7 days
- Status updates as the issue is reproduced and triaged
- Coordinated disclosure after a fix is available
